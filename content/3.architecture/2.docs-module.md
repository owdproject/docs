This page is a **directional spec** — update it when the module actually lands in a monorepo.

## UX goals

- On the OWD desktop, opening **`/docs`** (or `/documentation`, if configurable) should feel like **first-class** documentation, not a foreign site.
- **Chrome** (title bar, shadows, borders, theme fonts) should match the **active theme** so docs do not look “pasted in”.
- **Standalone**: the **`owd-docs`** repo remains valid on its own for authoring and classic site deploys.

## Technical principles

### Single content corpus

- Recommended source of truth: **`content/`** (Nuxt Content) in **`owd-docs`**, or an npm package that **embeds** that corpus at build time.
- Avoid two diverging copies (static site vs desktop): CI can publish a **content bundle** consumed by both the public site and the desktop module.

### Nuxt module (desktop side)

1. New package, e.g. **`@owdproject/docs`**:
   - depends on **`@nuxt/content`** (or pre-generated HTML only if you choose that route);
   - registers **`/docs/**`** routes under the desktop root;
   - registers a **minimal layout**: scroll region + readable width; **no** OS-specific chrome in the module itself.

### Theme integration

- The **theme** supplies visual **wrapper**: maximisable window, chrome, optional theme sidebar.
- Suggested contract:
  - the docs module exposes a **root page** and internal navigation (TOC, optional search);
  - the theme **wraps** with its **`Window`** / desktop layout or future core **slots** (`docs:before`, `docs:after`).
- **Core** stays agnostic: no PrimeVue dependency inside the docs module itself; the theme may still inject CSS variables.

### Search

- Standalone Docus provides full-text search.
- When embedded: **optional** (client-side index, Nitro endpoint, or disabled in v1).

### SPA vs SSR

- OWD desktop is often **`ssr: false`**: Content must work in that mode (client navigation, prefetch).
- If you later add SSR for SEO on **`/docs`**, decide at **deploy** level for the desktop — not mandatory for v1 embed.

## Implementation checklist (when coding)

- [ ] Package **`@owdproject/docs`** (final name TBD) with **`defineNuxtModule`**
- [ ] **`/docs`** route mounting the Content renderer
- [ ] Smoke test with **`@owdproject/theme-win95`**: window / layout coherence
- [ ] Document enabling the module in **`desktop.config.ts`** (`modules: ['@owdproject/docs']`)
- [ ] CI pipeline: same **`content/`** for standalone site and module
