The **`playground/`** folder is a **Nuxt application** used only in development.

## Purpose

- Load **`@owdproject/core`** like the real desktop.
- Include your module via workspace link for hot reload on source changes.
- Validate **`nuxt prepare`** and types without publishing to npm.

## Minimal setup

- **`nuxt.config.ts`**: `modules: ['@owdproject/core']`, often `ssr: false` to match the desktop.
- **`desktop.config.ts`**: `defineDesktopConfig({ theme: '...', apps: ['@scope/my-app'] })` — the package name must resolve (pnpm **`workspace:*`**, npm, or **`file:`**). See [Package linking](/setup/package-linking).

## Commands

```bash [Terminal]
cd playground
pnpm install
pnpm run dev
```

Or use a root **`dev`** script that runs `nuxt dev playground`.

## Limits

The playground does **not** replace end-to-end testing on **`client/desktop`** when you need interactions with other modules or custom themes. For that, wire the app into **`desktop`** and run `pnpm run dev` from the client root.
