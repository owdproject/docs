## Local build

From the app package root:

```bash
pnpm run prepack
# or, for stub during development:
pnpm run dev:prepare
```

- **`prepack`** runs **`nuxt-module-build build`** → **`dist/module.mjs`** + types.
- **`dev:prepare`** — stub + playground **`nuxt prepare`** for daily dev.

## What to publish

- **`files`**: typically **`["dist"]`** (or **`["dist", "src"]`** if you ship development exports — match your theme/app convention).
- **`README`**: install via **`desktop add my-app`**, **`peerDependencies`** on **`@owdproject/core`**, minimal **`desktop.config.ts`** snippet.
- **Do not publish** `playground/` launch plugins — they are dev-only.

## Versioning

Follow **SemVer**. Breaking changes to public module exports or **`ApplicationConfig`** → **major** bump + changelog.

## Consuming from the client monorepo

```json
// desktop/package.json
"@owdproject/my-app": "workspace:*"
```

```ts
// desktop.config.ts
apps: ['@owdproject/my-app']
```

After editing app source:

```bash
pnpm run prepare:apps   # or app’s dev:prepare
pnpm run dev
```

Standalone repos should use **npm semver** on core — see [Package linking](/setup/package-linking).

## Validation

```bash
desktop validate .
```

Runs playbook checks before publish:

| Area | Examples |
|------|----------|
| Structure | `src/module.ts`, `src/runtime/app.config.ts`, `src/runtime/plugin.ts` |
| Plugin contract | `defineDesktopApp`, `owd-*-register` name, server guard |
| Playground | `package.json` depends on your package + `@owdproject/core` |
| Optional | `launch-*.client.ts`, GitHub Pages workflow |

Fix errors before `npm publish`. Warnings (missing launch plugin) are optional but recommended for DX.
