## Window models (`windows`)

Each key under **`windows`** is a **model** (e.g. `main`, `settings`). For each model you typically set:

- **`component`**: `() => import('./components/Window/WindowMain.vue')` (dynamic import).
- **`size`**, **`position`**, **`resizable`**, **`minimizable`**, … — see `WindowConfig` types in core.

Open a window from **`commands`** with `app.openWindow('main')` after wiring the command.

## Entries

**`entries`** describe **how the user starts** a flow (app menu, search, etc.):

```ts
entries: {
  main: { command: 'open' },
  about: { command: 'about' },
}
```

Each entry has a **`command`** that must exist under **`commands`** (or be handled by the shell / launcher).

## Commands

Keys under **`commands`** are functions `(app: IApplicationController, ...args) => void | Promise<void>` that call **`openWindow`**, update state, etc.

Minimal example:

```ts
commands: {
  about: (app) => {
    app.openWindow('main')
  },
}
```

## Terminal commands (optional)

If you expose text commands to the OWD shell, the core wires **`TerminalManager`** to registered apps. Register command names during app init (see `ApplicationController` / application store).

## Singleton

With **`singleton: true`** you avoid multiple instances when your product model requires a single copy (e.g. one “About” app).
