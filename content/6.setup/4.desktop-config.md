Place **`desktop.config.ts`** next to the **Nuxt root** of your desktop project (alongside **`nuxt.config.ts`**). In the client monorepo that is **`desktop/desktop.config.ts`**.

::callout{icon="i-lucide-history"}
**Legacy filename:** **`owd.config.ts`** is still resolved for older projects. New projects should use **`desktop.config.ts`**.
::

## Minimal shape

```ts
import { defineDesktopConfig } from '@owdproject/core'

export default defineDesktopConfig({
  theme: '@owdproject/theme-nova',
  apps: ['@owdproject/app-about'],
  modules: [],
})
```

## Main fields

| Field | Meaning |
|-------|---------|
| **`theme`** | Theme Nuxt module (npm package name). Required for a predictable shell. |
| **`modules`** | Extension modules loaded **after** the theme, **before** **`apps`** (e.g. **`@owdproject/module-fs`**, **`@owdproject/module-persistence`**). |
| **`apps`** | App modules; each registers programs via **`defineDesktopApp`**. |

## Shell keys (merged into runtime config)

Keys other than **`theme`**, **`apps`**, and **`modules`** are **shell configuration** only. They merge into:

- **`runtimeConfig.public.desktop`**
- **`appConfig.desktop`** (for runtime overrides via **`useDesktopManager().setConfig()`**)

They are **never** spread onto **`_nuxt.options`** — put Nuxt options in **`nuxt.config.ts`**.

Common shell keys:

| Key | Purpose |
|-----|---------|
| **`name`** | Shell id (CSS modifier e.g. `owd-desktop--nova`). |
| **`systemBar`**, **`dockBar`** | Shell chrome toggles and layout. |
| **`workspaces`** | Multi-desktop / overview behaviour. |
| **`explorer`** | Quick access, special folders, mount labels (with **`module-fs`** + theme explorer UI). |
| **`defaultApps`**, **`features`** | Feature → default app routing. |
| **`docs`** | Future docs module integration. |

Unknown keys trigger a **dev warning**. Keys that look like Nuxt config (**`ssr`**, **`vite`**, …) are rejected with a hint to use **`nuxt.config.ts`**.

## Merge semantics

Core and themes use **[defu](https://github.com/unjs/defu)** (UnJS):

- **User / existing `desktop.config` values win**; theme module **`defaults`** fill **`undefined`** only.
- **`useDesktopManager().setConfig()`** uses the same model so **`DesktopCore`** props override at runtime.

Themes should **not** double-merge with redundant objects or force-assign nested keys after merge — set **`defaults`** on **`defineNuxtModule`** instead.

## Validation

**`@owdproject/core`** validates the default export after load:

- **`theme`**: string (npm package name) or omitted.
- **`apps`** / **`modules`**: arrays of strings when present.

Malformed shapes throw **before** theme or app modules install.

## Errors

Missing or invalid config throws with **path + remediation**. Syntax errors surface from the bundler or validation step.

## Example with filesystem

```ts
export default defineDesktopConfig({
  theme: '@owdproject/theme-nova',
  apps: ['@owdproject/app-about'],
  modules: [
    '@owdproject/module-fs',
    '@owdproject/module-persistence',
  ],
  workspaces: { enabled: true },
})
```

Install modules first: **`desktop add module-fs`**.
