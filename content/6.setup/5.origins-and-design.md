## Why it exists

**Open Web Desktop** targets a **browser desktop** that is modular: themes resembling real OS shells, **apps** as Nuxt packages, and a **core** that does not privilege one visual style. The goal is to lower the barrier to shipping your own web desktop as **open source**.

## Core technical choices

1. **Nuxt 4 + Vue 3** — installable modules, SSR optional (desktop is often SPA).
2. **Nuxt modules everywhere** — themes, apps, extensions share **`defineNuxtModule`**.
3. **Pinia + composables** — reactive shell and application state.
4. **PrimeVue + Tailwind** — consistent UI; themes customize presets and SCSS.
5. **Declarative config** — **`desktop.config.ts`** as the project contract.
6. **defu merge** — user shell config wins; theme defaults fill gaps.
7. **Kits + external modules** — explorer UI and VFS stay out of core proper.

## What “agnostic core” means

**Core** ships **`Desktop*`** kernel components and window manager APIs — not Win95 assets or explorer trees. Each “OS” is a **theme**. Filesystem and dialogs delegate to **`kit-fs`**, **`kit-theme`**, and **`module-fs`**.
