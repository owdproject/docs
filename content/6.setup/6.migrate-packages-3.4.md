This page guides you through upgrading themes, apps, and extension modules from **OWD Core 3.3.x** to **3.4.x**. 

Version 3.4 cleans up the core kernel by removing direct presentation dependencies (PrimeVue, Tailwind CSS) and moving them into a decoupled, pluggable kit system.

---

## Agent / Contributor Checklist

| Step | Target | Action |
|------|--------|--------|
| **1** | All | Bump **`peerDependencies['@owdproject/core']`** to **`^3.4.0`** (or `workspace:*` in the monorepo). |
| **2** | Themes | Install **`@owdproject/kit-primevue`** as a peer/dependency. Replace `@owdproject/kit-theme` / `kit-explorer` / `kit-fs` imports with kit-primevue / core runtime paths. |
| **3** | Themes | `module.ts`: call `await installModule('@owdproject/kit-primevue')` (if PrimeVue is needed) and use `registerThemeTailwindPath(nuxt, import.meta.url)` from kit-tailwind. |
| **4** | Apps | If using Tailwind classes, import `registerTailwindPath` from `@owdproject/kit-tailwind/kit/registerTailwindPath` instead of `@owdproject/core`. |
| **5** | All | Update imports of `defineDesktopApp` to `@owdproject/core` (previously imported from `runtime/utils/utilDesktop`). |
| **6** | All | Update imports of `defineDesktopModule` / `defineDesktopTheme` to `@owdproject/core/kit/authoring`. |
| **7** | All | If using filesystem features, import headless stores and composables (`useExplorerStore`, `useExplorerWindow`) from `@owdproject/module-fs/runtime/...` and UI components (`DesktopExplorer*`) from `@owdproject/kit-primevue`. |

## 1. PrimeVue & Tailwind Out of Core

Starting with 3.4, `@owdproject/core` no longer bundles or installs `primevue` or `@nuxtjs/tailwindcss` by default. 

### For Themes:
Demo and PrimeVue-based themes must explicitly pull in `@owdproject/kit-primevue` (previously `@owdproject/kit-theme` / `kit-explorer` / `kit-fs` which are now deprecated):

**Old `module.ts`**:
```ts
import { defineNuxtModule } from '@nuxt/kit'
import { registerTailwindPath } from '@owdproject/core'

export default defineNuxtModule({
  async setup(options, nuxt) {
    await installModule('@owdproject/kit-theme')
    registerTailwindPath(nuxt, resolve('./runtime/components/**/*.{vue,mjs,ts}'))
  }
})
```

**New `module.ts`**:
```ts
import { defineDesktopTheme } from '@owdproject/core/kit/authoring'
import { registerThemeTailwindPath } from '@owdproject/kit-tailwind/kit/registerTailwindPath'

export default defineDesktopTheme({
  async setup(options, nuxt) {
    await installModule('@owdproject/kit-primevue')
    
    // Automatically registers components/ and pages/
    registerThemeTailwindPath(nuxt, import.meta.url)
  }
})
```

### For Apps using Tailwind:
Apps that define utility classes must import `registerTailwindPath` from kit-tailwind:

```ts
// src/module.ts
import { registerTailwindPath } from '@owdproject/kit-tailwind/kit/registerTailwindPath'

registerTailwindPath(nuxt, resolve('./runtime/components/**/*.{vue,mjs,ts}'))
```

## 2. Decoupled Explorer & VFS

All virtual filesystem and explorer code has been decoupled:
* **Headless logic** (ZenFS virtual filesystem, tabs, stores, DnD operations) lives in `@owdproject/module-fs`.
* **PrimeVue UI** (workspace explorer view, toolbars, list renderers) lives in `@owdproject/kit-primevue`.

### Import Mapping:

| Old Path | New Path |
|----------|----------|
| `@owdproject/kit-theme/runtime/composables/*` | `@owdproject/core/runtime/composables/*` |
| `@owdproject/kit-fs/runtime/*` (headless) | `@owdproject/module-fs/runtime/*` |
| `@owdproject/kit-explorer/runtime/composables/*` | `@owdproject/module-fs/runtime/composables/*` |
| `@owdproject/kit-explorer/runtime/components/*` | `@owdproject/kit-primevue/runtime/components/explorer/*` |
| `@owdproject/kit-fs/runtime/components/*` | `@owdproject/kit-primevue/runtime/components/explorer/*` |
| `createDesktopDialogs` (from kit-theme) | `@owdproject/kit-primevue/runtime/dialogs/createDesktopDialogs` |

## 3. Pinia Store ID Renames

Pinia stores have been renamed to drop the `owd/` prefix in favor of `desktop/` to match runtime identifiers.

| Old Store ID | New Store ID |
|--------------|--------------|
| `owd/desktop` | `desktop` |
| `owd/desktop/workspace` | `desktop/workspace` |
| `owd/desktop/window` | `desktop/window` |
| `owd/desktop/volume` | `desktop/volume` |
| `owd/desktop/defaultApps` | `desktop/defaultApps` |
| `owd/application/${appId}/windows` | `desktop/application/${appId}/windows` |
| `owd/application/${appId}/meta` | `desktop/application/${appId}/meta` |

> [!NOTE]
> If `@owdproject/module-persistence` is installed, persisted state is migrated automatically on the first client boot. This change is only breaking if you have custom scripts or tools that query raw `localStorage` keys directly.

## 4. Verification

After migrating, verify your package builds and runs:

```bash
# Inside the package directory
pnpm exec nuxt-module-build build --stub

# Inside playground or desktop client
pnpm exec nuxt prepare
pnpm dev
```
Check that:
1. No console warnings appear for deprecated keys or missing exports.
2. Extension configs merge correctly.
