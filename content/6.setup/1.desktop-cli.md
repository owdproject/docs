The **`desktop`** command ships with **`@owdproject/core`** (`packages/core/bin/desktop.js`). In the client monorepo, root **`package.json`** exposes it as **`pnpm desktop`**.

::callout{icon="i-lucide-info"}
**Legacy alias:** **`owd`** still works but is deprecated — prefer **`desktop`** in scripts and docs.
::

## Common commands

| Command | Effect |
|---------|--------|
| **`desktop add <name>`** | Resolves `@owdproject/<name>` (app, theme, or module), installs the package, updates **`desktop/package.json`**, and registers it in **`desktop.config.ts`**. |
| **`desktop add <name> --npm`** | Force install from the npm registry (skip Git clone). |
| **`desktop add <name> --from user`** | Clone from `github.com/user/<name>` instead of `owdproject`. |
| **`desktop add <name> --dev`** | Clone into the workspace (e.g. under `packages/` or ignored paths) for local development. |
| **`desktop validate [path]`** | Run the module/app/theme playbook checks (playground layout, exports, peers). |
| **`desktop template`** | Sync the **`npm create owd`** scaffold from the core blueprint. |

Examples:

```bash
desktop add app-todo
desktop add theme-nova
desktop add module-fs
desktop add module-persistence --npm
```

## Nx integration

In the official client repo, some install flows delegate to Nx targets (e.g. **`desktop:install-app`**) defined in **`desktop/project.json`**. Behaviour matches the table above; Nx is an orchestration layer, not a separate CLI.

## Prerequisites

- A Nuxt desktop project with **`modules: ['@owdproject/core']`** in **`nuxt.config.ts`**.
- **`desktop.config.ts`** (or legacy **`owd.config.ts`**) beside **`nuxt.config.ts`**.
- **pnpm** at the version pinned in **`packageManager`**.

## Manual alternative

When the CLI is unavailable or you use a custom layout:

1. **`pnpm add @scope/pkg`** in **`desktop/`** (or workspace root, depending on your layout).
2. Add the package name to **`desktop.config.ts`** under **`theme`**, **`modules`**, or **`apps`**.
3. For packages that ship **`dist/`**, run **`pnpm run dev:prepare`** in the package or **`pnpm run prepare:stubs`** from the client root.

See [Package linking](/setup/package-linking) for **`workspace:*`** vs npm vs Git URLs.
