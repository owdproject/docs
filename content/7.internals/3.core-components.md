Kernel Vue components live under **`packages/core/runtime/components/`** and register **globally** with **`pathPrefix: false`**. They are **structural** building blocks: window behaviour, workspace hooks, app mounting ŌĆö not OS-specific pixels.

::callout{icon="i-lucide-alert-triangle"}
**Breaking change (core 3.3):** components were renamed from **`Core*`** to **`Desktop*`** (`CoreDesktop` ŌåÆ **`DesktopCore`**, `CoreWindow` ŌåÆ **`DesktopWindow`**, ŌĆ”). Update theme templates when upgrading peers.
::

## Shell

| Component | Role |
|-----------|------|
| **`DesktopCore`** | Kernel shell wrapper: props, workspace init, default slot. Themes wrap this in **`Desktop.vue`**. |
| **`DesktopBackground`** | Optional full-screen background slot helper. |
| **`DesktopContent`** | Main desktop content region (apps + theme slots). |
| **`DesktopApplicationRender`** | Renders running application windows for a workspace filter. |

## Windows

| Component | Role |
|-----------|------|
| **`DesktopWindow`** | Drag, resize, focus, z-order events ŌĆö neutral chrome hooks. |
| **`DesktopWindowNav`** | Title bar / nav region. |
| **`DesktopWindowContent`** | Client area for app UI. |

Themes implement their own **`Window.vue`** that wraps **`DesktopWindow`** and supplies close/minimize/maximize visuals.

## What is *not* in core

Explorer file/folder UI, ZenFS, and **`useExplorerStore`** moved to **`@owdproject/kit-fs`**. Dialog implementation contracts live in **`@owdproject/kit-theme`**.

## Theme obligations

1. Provide **`Desktop.vue`** registered by the theme module.
2. Wrap kernel components; do not reimplement focus/z-order/workspace logic.
3. Use **`provide/inject`** for **`windowController`** only inside theme window wrappers.

Full contract: [Kernel contract](/internals/kernel-contract).
