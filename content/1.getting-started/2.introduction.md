## What is Open Web Desktop

**Open Web Desktop (OWD)** is a set of Nuxt packages for **desktop-like experiences in the browser**: windows, taskbars, workspaces, and installable **Nuxt modules** as applications. Visual style (“Windows 95”, “GNOME”, “Nova”, …) is a **theme**; apps stay API-compatible across themes.

The **engine** (`@owdproject/core` **3.3+**) keeps window lifecycle and shell contracts neutral. **Kits** (`kit-theme`, `kit-fs`, …) share dialogs and explorer UI. **Extension modules** (`module-fs`, `module-persistence`) install from npm when you need a virtual filesystem or persistence.

## Client monorepo vs standalone repos

- **`owdproject/client`** — core, kits, reference **`desktop/`**, CLI, template blueprint.
- **Theme / app repos** — separate GitHub repositories (e.g. **`theme-nova`**, **`app-about`**), usually consumed via **npm** or cloned into **`themes/*`** / **`apps/*`** for monorepo dev.

**Important:** mixing **`workspace:*`** with packages that are **not** in the pnpm workspace causes install failures. Read **[Package linking](/setup/package-linking)** before wiring **`package.json`**.
