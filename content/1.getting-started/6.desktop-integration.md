This page summarizes the **product goal**, not necessarily a shipped feature yet.

## Why

Today **`owd-docs`** is a **standalone** site (Docus / Nuxt UI). The desired experience is that on the OWD desktop, documentation is available under **`/docs`** (or similar) **inside** the active theme — window chrome and decorations match the simulated “OS” (Win95, future themes, …), not a foreign iframe.

## Technical direction (summary)

Details: **[Documentation module](/architecture/docs-module)**.

In short:

1. A package such as **`@owdproject/docs`** (working name): Nuxt module registering routes and layout.
2. The **theme** can wrap the docs view (chrome slots) or delegate to shared layouts.
3. A **single content corpus** in **`content/`** (or a built artifact) avoids two divergent sources between standalone site and desktop.

## What you can do now

- Write and review content in **`content/`** here.
- Keep concepts aligned with the **client** repo and **`AGENTS.md`**.

When the module ships, the same prose can be consumed from the desktop without rewriting everything.
